## ② PM 역할

**목표:** 팀원 모두가 *같은 방향을 정확히 이해하고 공유한 채로* 간다. PM(승준)은 그 **공유된 방향을 만들고 유지**한다.

**모자 둘:** *리걸*(법 → 구현 가능한 명세로 번역 + 법률 판단) + *PM*(결정·순서·병목·소통).
**안 하는 일:** 컨트랙트 코딩(준서·택범) · 발표 제작(승환) · 최종 법률 의견(보경).

| 대상 | PM이 주는 것 |
|---|---|
| 개발팀 (준서·택범) | 준법 검사 명세표 · 검증 모듈 세트 확정 · 수정요청서 |
| 보경 변호사 | 위임 질문지 · 검토 맥락 |
| 최승환 (팀장·발표) | 현황 대시보드 · 결정 공지 · 발표용 정리 |

*두나무 = Gasok(가속) 주최 외부사 + Giwa 제공자. 팀 아님(외부 맥락).*

## ③ 문서 지도 + 공유 기준

**원칙: 팀이 보는 건 *정본 소수*. 나머지는 내부 작업물(요청 시에만 공유).**

| 정본 문서 | 무엇 | 주 독자 | 위치 |
|---|---|---|---|
| **이 허브** | 현황·역할·지도 front door | 전원 | docs/pm/ |
| 준법 검사 명세표 | 법 → 검사 → 실패 처리 (구현 자재) | 개발팀 | docs/compliance/ |
| 검증 모듈 설명서 | 모듈별 상세(적격투자자·QP·내부자…) | 개발팀·보경 | docs/compliance/elements/ |
| 설계 결정 기록(ADR) | 주요 결정 + 근거 | 전원 | docs/decisions/ |
| 위임 질문지 | 변호사에 던지는 질문 | 보경 | (위임 letter) |
| 회의자료 | 회의 안건·공유 | 전원 | _PM/ |

**무엇을 · 누구에게 · 언제 공유하나:**
- *결정이 닫히면* → 설계 결정 기록(ADR) 작성 + 이 허브 갱신 → 전원 공지
- *개발 자재가 나오면* → 명세표·설명서 → 개발팀
- *법률 질문이 쌓이면* → 위임 질문지 → 보경 변호사
- *큰 흐름·진행 상황* → 이 허브 (매 회의 갱신)
- **내부 작업물**(분석 노트·전략 초안 등)은 *요청 시에만*
