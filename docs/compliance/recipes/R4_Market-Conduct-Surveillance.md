# R4 — 시장행위 감시 (Recipe 명세 = 역방향 금지규범 + 패턴 C 우위)

> **이 문서는 무엇인가.** Recipe = *하나의 법률효과를 논증.* 그런데 R4는 **R1·R2·R3와 논리가 *뒤집혀 있다.*** 면제 recipe(R1~R3)는 *"요건을 충족하면 → 면제 효과가 *성립*한다"*(조건→효과). **R4는 *금지규범*이다 — *"금지행위에 해당하면 → 위반"*.** 즉 *조건 충족을 증명*하는 게 아니라 *금지 저촉을 차단·감시*한다.
> **그리고 결정성이 약하다.** 시세조종(§9(a))·사기(§10(b)·§17(a))는 *고의·기만(scienter)*이 요건 — *기계가 위반을 단정할 수 없다.* 따라서 R4의 본체는 **패턴 C(객관 패턴 탐지 → flag → 사람 판단)**이고, *결정론 차단(패턴 A)*은 *Reg M 기간·운영자 자기거래*에 한정된다.
> **그래서 "법률효과"도 약하다.** R4가 보장하는 건 *"시장행위 위반이 부존재한다"*가 **아니라** *"결정론 금지(Reg M·자기거래)에 저촉되지 않았고, 의심 행위는 감시·표시되어 사람 검토로 넘어간다"*이다. 이 *경계*가 R4의 핵심.
> **자산 일반성(ADR-006):** R4는 *시장(venue) 단위* 규범 — **가장 자산 무관**하다. 시장 무결성은 *모든 자산에 보편*. 자산별 값 거의 불요(Reg M 기간만 R1 발행상태에서 옴).

## §1. 법률효과 + 법적 구조 (역방향)

**효과(소결론): "이 거래는 *결정론 금지*(Reg M 매수 제한·운영자 자기거래)에 저촉되지 않았고, *조작·사기 의심*은 감시·표시되었다."**

```
면제 recipe(R1~R3):   [요건 충족]  ──→  [면제 효과 성립]          (정방향)
금지 recipe(R4):       [금지행위 해당] ──→ [위반·차단/flag]         (역방향)

R4의 두 층:
  ① 결정론 금지 (패턴 A — hard block)
      Reg M(101/102) 판매 중 매수 · 운영자 자기거래
      → 거래 *직전 차단* (날짜·신원으로 판정 가능)
  ② 감시 (패턴 C — flag + 사람)
      시세조종(§9(a)) · 조작·기만(§10(b)/10b-5) · 사기(§17(a))
      → *고의 요건* → 기계 단정 불가 → 객관 패턴 탐지·표시, 위반 판단은 사람
```

## §3. ① 법률효과 — 무엇을 "보장"하는가(그리고 무엇을 *못* 하는가)

- ✅ **보장:** Reg M 제한기간 중 발행자·관여자 매수 *차단*, 운영자 자기거래 *차단*(결정론). 조작·사기 의심 패턴 *탐지·표시·기록*.
- ❌ **보장 안 함:** *"이 거래가 사기가 아니다"의 확정.* 고의·기만은 *off-chain 사람 판단* 영역 — R4는 *증거 표시*까지. (§9 결정론 경계)

## §4. ② 법률 논증 — 금지규범 조문별

> **읽는 법.** §4.0에서 *왜 시장행위 규범이 모든 거래에 얹히는가*를 세우고, §4.1(Reg M)·§4.5(자기거래)는 *결정론 차단(패턴 A)*, §4.2~§4.4(§9(a)·§10(b)·§17(a))는 *감시(패턴 C)*로 조문별 논증한다. **삼단논법의 *방향이 역*임에 유의** — "금지 저촉 시 위반".

### §4.0 문제 설정 — 시장 무결성은 모든 거래에 적용

> 시장 조작·사기 금지는 *면제와 무관하게* 모든 증권거래에 적용되는 *상시 규범.* 발행(R1)·전매(R2)·펀드(R3) *어느 경로든* 거래 자체가 조작·사기면 위반. → R4는 *모든 거래에 곱해지는* 시장행위 층.

---

### §4.1 Reg M — 판매 중 매수 제한 (결정론, 패턴 A) → **F-04**

> **17 CFR §242.101**(요지): distribution 참여자(underwriter 등)는 *restricted period 동안* 대상증권을 *bid·purchase 금지.* **§242.102**: *발행자·매도 증권보유자·affiliated purchaser*도 동일 금지.

- **대전제:** *판매(distribution) 제한기간 중* 발행자·관여자는 그 증권을 *사면 안 된다*(가격 인위 부양 = 조작).
- **소전제(부품):** **F-04** — *제한기간(날짜 윈도우)* ∧ *매수자 신원(발행자/관여자)* 으로 *결정론 판정* → 해당 시 매수 차단.
- **소결론:** ∴ F-04가 (제한기간 ∧ 관여자 매수)면 BLOCK ⟹ Reg M 미저촉. *날짜·신원은 결정 가능 → 패턴 A.*

> Reg M 제한기간은 *R1 발행 상태*에서 온다(발행 진행 중 = 제한기간). → R-XJ/R1과 연동(F-04 frontmatter).

### §4.2 §9(a) — 시세조종 (감시, 패턴 C) → **F-02**

> **15 U.S.C. §78i(a)**(요지): *wash sale*(소유권 변동 없는 거래 §9(a)(1)(A))·*matched order*·*일련의 거래로 거래활발 *외관* 조성*(§9(a)(2)) 등 *조작 목적* 행위 금지.

- **대전제:** *거래활발 외관·가격 조작*을 *목적*으로 한 wash trade·matched order는 위반.
- **소전제(부품):** **F-02**(STATEFUL) — *객관 패턴*(동일·연계 계정 간 반복, 소유권 무변동 의심, spoofing/layering 주문패턴)을 *탐지·표시.* **단 *목적·고의*는 기계가 못 판정** → flag만.
- **소결론:** ∴ F-02가 패턴 탐지 시 *flag + 사람 검토* ⟹ §9(a) *감시 충족*(위반 *확정*은 사람). **패턴 C.**

### §4.3 §10(b)/Rule 10b-5 — 조작·기만 (감시, 패턴 C) → **F-02·F-03**

> **15 U.S.C. §78j(b)** + **17 CFR §240.10b-5**(요지): 증권 매매와 관련해 (a) 사기 *scheme*, (b) *중요사실 허위·누락*, (c) *사기·기만으로 작용하는 행위* 금지.

- **대전제:** 매매 관련 *기만적 device·scheme*은 위반.
- **소전제(부품):** **F-02**(가격 조작 측면)·**F-03**(기만·허위 측면)이 *이상신호 탐지·표시.* *기만·중요성·고의*는 *사람 판단.*
- **소결론:** ∴ F-02·F-03 flag + 사람 ⟹ §10(b) *감시 충족.* **패턴 C.**

### §4.4 §17(a) — 발행·판매 사기 (감시, 패턴 C) → **F-03**

> **15 U.S.C. §77q(a)**(요지): 증권의 *offer·sale*에서 (1) 사기 device, (2) 허위진술로 금전취득, (3) 사기·기만으로 작용하는 거래·관행 금지.

- **대전제:** *판매 단계*의 기만·허위는 위반.
- **소전제(부품):** **F-03**(STATEFUL) — 허위·이상 자금흐름·기만 신호 *탐지·표시·처리기록.*
- **소결론:** ∴ F-03 flag + 사람 ⟹ §17(a) *감시 충족.* **패턴 C.**

### §4.5 운영자 자기거래 — §10(b) + ICA §17 (결정론, 패턴 A) → **F-01**

> **15 U.S.C. §78j(b)**(이해상충 남용) + **15 U.S.C. §80a-17**(ICA §17 — 펀드 affiliate 거래 제한)(요지): *운영 주체가 자기 시장에서 이용자와 맞붙거나 front-run* 하는 자기거래·이해상충 금지.

- **대전제:** *운영자(및 임직원·계열)*는 *자기가 운영하는 시장에서* 자기거래·front-running 금지.
- **소전제(부품):** **F-01** — *운영자 측 신원*으로 *결정론 식별* → 운영자 주문이 이용자와 충돌하면 차단.
- **소결론:** ∴ F-01이 운영자 자기거래 식별 시 BLOCK ⟹ 미저촉. *신원은 결정 가능 → 패턴 A.*

### §4.6 Gap·경계 분석 (4분류 + 결정성)

| 이슈 | 분류 | 처리 |
|---|---|---|
| 조작·사기의 *고의/기만* 판정 | **(b)** | *법적·사실 판단* → 기계 불가. F-02/F-03은 *패턴 탐지·표시*까지, 위반 확정은 **사람(off-chain)** |
| Reg M 제한기간 산정 | **(a→결정)** | 발행 상태(R1)에서 윈도우 입력 → F-04 결정론 |
| 운영자 신원 명부 | **(a)** | `OperatorRegistry`(시스템 명부) → F-01 결정론 |
| 감시 임계·룰 튜닝 | **(c/운영)** | flag 임계는 *운영 파라미터* — 코드 상수 아님, 거버넌스로 조정 |

→ **결론: R4는 *결정론 차단 2개(F-04·F-01, 패턴 A)* + *감시 표시 2개(F-02·F-03, 패턴 C)*.** 신규 부품 불요. **핵심은 *경계 명시* — R4의 보장은 "차단+감시"이지 "위반 부존재 확정"이 아님.**

## §5. ③ Activation Logic

- **모든 거래**에 always-on(venue 단위, 자산 무관). `fundExemption`·`issuanceFramework`와 무관하게 *항상* 실행.
- F-04(Reg M)만 *R1 발행 제한기간*일 때 활성 — 그 외엔 자기거래·감시만.

## §6. ④ Composition — 차단층 AND + 감시층 flag

```
R4(@매 거래):
  ① 결정론 차단 (패턴 A, hard block):
       PASS_block ⟺ ¬F-04.prohibited(제한기간∧관여자매수) ∧ ¬F-01.selfDeal(운영자충돌)
  ② 감시 (패턴 C, 비차단 원칙):
       F-02.scan(시세조종 패턴) → flag? → off-chain 사람 검토
       F-03.scan(사기 신호)     → flag? → off-chain 사람 검토 + 처리기록(STATEFUL)

  R4_PASS(tx) ⟺ PASS_block ∧ [F-02·F-03 스캔 *수행*됨(필요 시 flag emit)]
              # 주의: 감시층 PASS = "스캔 수행+표시"이지 "위반 없음 확정"이 아님
```
- **R1~R3와 차이:** 면제 recipe는 *PASS = 효과 성립(확정).* **R4의 감시층 PASS = *검사 수행*(확정 아님)** — 위반 판정은 사람. *이 비대칭이 R4의 정체성.*
- **차단층(F-04·F-01)**만 *단일 거래를 막는다.* 감시층(F-02·F-03)은 *원칙적으로 비차단*(거래 흐름 유지 + 사후 검토) — 단 거버넌스가 정한 *hard 임계* 초과 시 차단 가능.

---

## §7. ⑤ 거절·예외 처리 (차단 vs 표시 분리)

- **차단(패턴 A):** `R4_BLOCK_REGM`(F-04)·`R4_BLOCK_SELFDEAL`(F-01) → *거래 거부* + 사유.
- **표시(패턴 C):** `R4_FLAG_MANIPULATION`(F-02)·`R4_FLAG_FRAUD`(F-03) → *거래는 진행*(원칙), *flag 기록* → 사람 검토 큐. STATEFUL 이력 누적.
- *오탐(false positive)* 대비: 감시는 *비차단 기본* — 정상 거래를 기계가 막지 않도록. 차단은 *결정론 확실 영역*만.

---

## §8. ⑥ Conflict·Interaction — R4는 모든 거래에 곱

| 상대 | 패턴 | 설명 |
|---|---|---|
| **R1·R2·R3** | **Cumulative(항상)** | 발행·전매·펀드 *어느 경로든* 시장행위 규범 적용. R4는 *모든 거래*에 R1/R2/R3와 함께 실행 |
| **R-XJ** | **Cumulative(always-on)** | F-04 Reg M·자기거래·감시 모두 상시. A-01·A-02도 |
| 면제 recipe 일반 | **직교(층 다름)** | 면제 = "발행/전매/펀드 *자격*", R4 = "거래 *행태*" — 다른 관심사. PASS 조건 독립 |

> **핵심:** R4는 *자격(R1~R3)과 무관한 행태 규범.* 적격 매수인(QP/AI)이 적법 경로로 거래해도 — *그 거래 방식*이 wash trade면 §9(a) 위반. 그래서 R4는 *모든 거래에 항상 곱해지는* 행태 검문.

## §9. 📐 결정론 경계 (R4에서 가장 중요)

| ✅ 온체인(결정론·차단) | 🔵 off-chain(claim·사람 판단) |
|---|---|
| Reg M 제한기간×관여자 신원(F-04)·운영자 자기거래 식별(F-01)·감시 패턴 탐지·flag emit·STATEFUL 이력 | **조작·사기의 *고의/기만* 판정(F-02·F-03)** · flag의 위반 여부 확정 · 중요성(materiality) 판단 |

> **R4의 정체성:** *온체인은 "패턴을 본다", off-chain은 "위반을 정한다."* 기계가 *scienter를 단정하면 오히려 위험* — 정상 거래 차단·법적 과신. R4는 *경계를 지켜* 결정론은 차단, 판단은 사람에 둔다.

## §10. 자산 적용 (BUIDL = 예시)

- BUIDL을 포함 *모든 자산* 거래에 R4 동일 적용 — 시장 무결성은 보편(자산별 분기 거의 없음).
- BUIDL 발행 제한기간엔 F-04로 발행자측 매수 차단. 평시엔 F-01 자기거래 차단 + F-02/F-03 감시.
- *다른 자산*도 코드 동일 — R4가 *가장 자산 무관*한 recipe(ADR-006 강한 사례).
